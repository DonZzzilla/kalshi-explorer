💡 What is Kalshi?

Kalshi is a prediction market where you trade on the outcome of real-world events. Each market has a YES and NO price in cents (1-99). If you buy YES at 65¢ and the event happens, you get $1.00 per contract. If not, you lose your 65¢.

Example: "Dow Above 45000" — YES price is 12¢
This means the market thinks there's a 12% chance the Dow closes above 45000.
Buy 100 YES contracts at 12¢ each = $12 invested. If Dow > 45000, you get $100. Profit = $88.

💡 Probability Calculator

This tool shows you exactly what you'd earn at different buy prices and investment amounts. Kalshi contracts pay out $1.00 if you're right, $0 if you're wrong. The buy price (in cents) is the market's estimate of probability.

How it works:
Buy price: 65¢ — Invest $100 — You get 153 contracts ($100 / $0.65)
If you win: 153 × $1.00 = $153 payout — Net profit = $53 — ROI = 53%
If you lose: $0 payout — Net loss = $100

💡 Earnings Projections

This shows your potential earnings at different probability levels, assuming you invest $100 per trade and make 4 trades per month. The chart helps you visualize which probability ranges offer the best risk/reward.

Reading the chart:
A market with 70¢ YES has 30¢ NO. If you buy NO at 30¢:
Win rate: 30% of the time you win $0.70 per $0.30 invested (133% ROI)
Loss rate: 70% of the time you lose your $0.30
Expected value per trade: (0.30 × $0.70) - (0.70 × $0.30) = $0

💡 System Architecture

This tool is built as a client-side web app that connects to Kalshi's public API. Here's how the data flows from the exchange to your screen.

🏢 DATA PIPELINE how it works Data PipelineMarkets flow from Kalshi's API through a CORS proxy (since browsers block direct cross-origin requests), get parsed into a standard format, analyzed for edge/opportunity, and rendered in the UI. When the API is blocked, demo data is used instead.

🧠 ANALYSIS ENGINE what it does Analysis EngineProcesses raw market data to find trading opportunities. Calculates implied probability, edge, volume-weighted confidence, and market microstructure signals.

  • • Implied probability from prices
  • • Edge: |YES - NO| = confidence
  • • Volume-weighted scoring
  • • Market microstructure analysis
  • • Historical price tracking

📡 API & CACHING why CORS? CORSCross-Origin Resource Sharing. Browsers block requests to external APIs unless the server allows it. Kalshi's API doesn't set CORS headers for browsers, so we use a proxy relay. The tool also caches data for 5 minutes to reduce API calls.

  • • Kalshi REST API (public)
  • • CORS proxy relay
  • • 5-min client cache
  • • WebSocket for live updates
  • • Demo data fallback

📝 API ENDPOINTS

GET /v2/markets — List all markets
GET /v2/markets/{ticker} — Single market detail
GET /v2/events — List all events
GET /v2/events/{event_id}/markets — Markets for an event

Base URL: https://external-api.kalshi.com
Auth: None required (public API)
Rate limit: ~100 req/min
