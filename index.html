💡 What is Kalshi? TEENAGER

People bet coins on if stuff happens. Sometimes yes, sometimes no. Win coins if right.

Kalshi is a website where grown-ups bet on whether things will happen. Like "will it rain tomorrow?" You pick YES or NO. If you're right, you win money. If you're wrong, you lose what you bet.

Kalshi is a prediction market where you trade on the outcome of real-world events. Each market has a YES and NO price in cents (1-99). If you buy YES at 65¢ and the event happens, you get $1.00 per contract. If not, you lose your 65¢.

Kalshi is a CFTC-regulated Designated Contract Market (DCM) and the first federally licensed exchange for event contracts in the US. Each contract is a binary option: pays $1.00 if the specified event resolves YES, $0.00 if NO. Prices are quoted in cents (1-99) representing the market's implied probability of YES — a 65¢ YES implies a 65% chance the market-makers collectively assign. Spreads, depth, and order books are visible via the API at /v2/markets/{ticker}/orderbook.

Example: "Dow Above 45000" — YES price is 12¢
This means the market thinks there's a 12% chance the Dow closes above 45000.
Buy 100 YES contracts at 12¢ each = $12 invested. If Dow > 45000, you get $100. Profit = $88.

📊 TOP 10 BY EDGE what is edge? Edge How much sure people are. How confident the market is. Big gap = very sure. Small gap = could go either way. The difference between YES and NO prices. A 90¢ YES / 10¢ NO market has an edge of 80¢ — meaning the market is very confident. Small edges (like 5¢) mean the outcome is uncertain. High edge = high confidence, but lower profit potential per trade. Edge = |YES_price − NO_price| in cents. A 50/50 market has 0¢ edge; a near-certain market approaches 100¢. Edge is inversely related to expected profit per contract — high-edge markets pay out less on a win because the YES price is already high. The most profitable trades are usually in the 60-80¢ edge band.

Two-input payout calculator using payout = floor(investment / price_cents) × 100 cents. The ROI formula is (100 − price) / price, expressed as a percent. Monthly/annual projections assume a fixed number of winning trades at a constant stake — these do not account for variance, fees (currently 1.5% on profits), or the fact that no edge persists indefinitely. Treat them as upper bounds for the geometric mean of independent identical bets.

💡 Earnings: What Could I Make? TEENAGER

Chart go up when right. Chart go down when wrong. Make money if right more.

This picture shows how much money you could win or lose at different prices. Cheap tickets win less money. Expensive tickets win more money, but you lose more often.

Imagine you find a market where the YES price is 70¢. That means you pay 70¢ per contract. If you're right, your contract pays $1.00. If you're wrong, you get nothing.

The earnings function is a piecewise linear payoff: profit_per_contract = (100 − price) cents on a win, −price cents on a loss. Expected value per contract is p × (100−p) − (1−p) × p = 0 under the null hypothesis that the market is correctly priced. To produce positive EV you need p̂ > p — i.e. your estimated probability must exceed the implied price. The breakeven win-rate at any price p is p / 100 (so 70¢ requires ≥70% accuracy to break even before fees).

Think of it like this:
• Price = what you pay for a ticket
• $1.00 = what the ticket pays if you win
• Profit = $1.00 minus what you paid
So at 70¢, every winning ticket makes 30¢ profit. But you lose your 70¢ on every losing ticket.

💡 How This Tool Works (Click Each Box!) TEENAGER

Boxes. One box talks to next box. Stuff comes out at end.

This page is like a row of helpers. The first helper asks Kalshi for all the markets. The next helper makes the messy answer nice and clean. The next helper picks out the good ones. The last helper draws the tables and charts for you to see.

This page connects to Kalshi's API, grabs market data, crunches the numbers, and displays them. Click any box in the pipeline below to learn what it does — explained simply.

The architecture is a five-stage data pipeline. Stage 1 hits Kalshi's public REST API (CORS forces a proxy in the browser-only deployment; a server-side bot can call directly). Stage 2 normalizes the response: tickers, prices, volume, and status. Stage 3 computes derived metrics — edge, mid-price, volume-weighted rank, and time-since-open. Stage 4 sorts and filters per the user's query. Stage 5 renders the views: market list, top-10 table, detail panel, and chart. Each stage is stateless and pure (modulo caching), so the same pipeline can be re-driven by a cron loop, a Discord slash command, or a Telegram bot with no rewrites.

You push button. Page ask far away box. Box send back stuff. Page show stuff.

When you click SCAN, the page asks Kalshi's computer for a list of all the markets. Kalshi sends back a long list. The page sorts it and shows you the most interesting ones first.

When you click SCAN, the page asks Kalshi's server for market data. Here's exactly what happens at each step, with the actual code.

Five-step REST round-trip: (1) fetch() with a query string to /v2/markets; (2) JSON parse into an array of market objects; (3) transform each object into a flat row of derived fields; (4) stable sort by a composite key (edge desc, volume desc); (5) DOM append into the result list. Network failures degrade to a cached or demo dataset; rate-limit responses (HTTP 429) trigger exponential backoff in the server-side variant.

🧠 Hermes Hub: How a 1-Man AI Agent Uses This TEENAGER

Tiny box in Don's house. Smart box watch Kalshi. Smart box tell Don when win.

Hermes is a computer program that lives on a small computer in Don's house. The computer is the size of a sandwich. Hermes wakes up every few minutes, looks at Kalshi, and if it finds something good, it sends Don a message.

This page is the front-end of a one-man AI trading research setup. The full system runs on a Raspberry Pi 4 with 2GB of RAM, hosted on Don's home network, and powered by Hermes Agent. It scans Kalshi 24/7, crunches probabilities, and alerts the user (the "master") via Discord or Telegram. Click each scenario below to see how it's wired together.

The deployment is a single Raspberry Pi 4 (2GB) running Raspberry Pi OS Lite. The stack: a systemd-supervised Python service polls Kalshi's public /v2/markets every 5 minutes; the analysis layer (the same code that drives this page) computes edge, volume, and recency scores; alerts are dispatched via a Discord webhook (push, no inbound port) or a long-poll Telegram bot. Memory footprint stays under 250MB because we never load the full market list into Python — we stream and filter as we go. The Pi 4's 4-core ARM Cortex-A72 is overkill for the workload; what matters is the always-on, low-wattage (~$3/yr) footprint. ⚙️ Quick-Start: From Zero to First Scan in 30 Minutes

  1. SSH into your Pi: ssh donzzz@raspberrypi.local
  2. Install deps: sudo apt install python3-pip sqlite3 -y
  3. Create the bot: mkdir ~/kalshi-bot && cd ~/kalshi-bot
  4. Write scan.py — a 30-line Python script using requests to hit https://external-api.kalshi.com/trade-api/v2/markets?status=open
  5. Test it: python3 scan.py — should print JSON
  6. Hook Discord: create a webhook in your server, paste URL into ~/.hermes/.env as DISCORD_WEBHOOK_URL=...
  7. Schedule it: crontab -e → */15 * * * * /usr/bin/python3 /home/donzzz/kalshi-bot/scan.py >> /home/donzzz/kalshi-bot/bot.log 2>&1
  8. Watch your Discord — first alert in 15 minutes 🚀
